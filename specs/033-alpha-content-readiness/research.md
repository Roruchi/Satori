# Research: Alpha Content and External Readiness

## Decision: Content follows stability

**Rationale**: Adding content before the spine is stable can hide broken progression, saves, or controls.

## Decision: One polished chain per useful family

**Rationale**: Alpha needs enough variety to be fun, not full catalog completion.

## Decision: Known issues are part of the release artifact

**Rationale**: Closed alpha can ship with gaps, but testers need to know which gaps are intentional.

## Decision: Polish the obvious alpha path before catalog breadth

**Rationale**: The full experience should work first. More spirits and structures are extra, while first ritual, Red Fox, Meadow dwelling, upgraded Fox Den migration with Red-Fox-only double Satori generation, Dew Bowl, Wind Chime, Mist Stag, Ku Seed, Void island separation, Chi+Ku calm-water island, and Suijin invitation must feel intentional.

## Decision: Include upgraded Fox Den, Dew Bowl, and Wind Chime as first structure sequence

**Rationale**: Fox Den makes housing explicit by moving Red Fox into a stronger home and granting double Satori generation for Red Fox only. Dew Bowl and Wind Chime then create a useful first-island loop without expanding into a broad catalog.

## Decision: Display build version in the menu

**Rationale**: Testers need a consistent place to read the exact build. Use zero-based SemVer with alpha prerelease and build metadata, for example `0.1.0-alpha+20260627.1`.

## Decision: No placeholders on the primary alpha path

**Rationale**: The alpha is allowed to be small, but the intended tester route must be polished enough for trust. Placeholder assets are allowed for non-primary content when that content is hidden, gated, or clearly outside the intended tester route.
