# Contract: External Alpha Readiness

## Purpose

Defines what must be ready before inviting closed alpha testers.

## Contract

External alpha readiness requires:

- included alpha content has clear purpose and no broken-looking gaps,
- included content is wired to Codex/save/load/tests where relevant,
- obvious path polish covers first ritual, Red Fox, Meadow dwelling, upgraded Fox Den migration with Red-Fox-only double Satori generation, Dew Bowl, Wind Chime, Mist Stag, Ku Seed, Void island separation, Chi+Ku calm-water island, and Suijin invitation,
- no placeholder art, audio, icon, or UI assets remain on the primary alpha path or release shell,
- build version is visible in the menu using `0.x.y-alpha+<build_id>` format,
- tester brief exists,
- known issues list exists,
- Web fresh-save playthrough reaches Suijin,
- Android fresh-save playthrough reaches Suijin.

## Validation

- Content audit completed.
- Tester handoff artifacts reviewed.
- Web and Android playthrough evidence recorded.
