# Implementation Plan: Phase 1 Seed Crafting in 3x3 Grid

**Branch**: `019-seed-crafting-grid` | **Date**: 2026-03-31 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/019-seed-crafting-grid/spec.md`

## Summary

Implement the first phase of grid crafting by moving seed crafting to a 3x3 token grid with position-insensitive recipe matching for one-token and two-token seed recipes only. Craft output goes to plant inventory, recipe tokens are consumed only on successful craft, successful craft clears only consumed slots, inventory-full blocks completion while preserving in-grid tokens, and mobile slot hit targets remain at least 48x48 px.

## Technical Context

**Language/Version**: GDScript on Godot 4.6  
**Primary Dependencies**: Existing crafting UI/menu scripts under `src/ui`, recipe/crafting logic under `src/seeds` and related gameplay services, GUT (`addons/gut`)  
**Storage**: Existing runtime/autoload state and current save flow; no new storage backend required in this phase  
**Testing**: GUT unit tests for deterministic recipe resolution and inventory-failure behavior, plus manual in-editor verification for UI interaction/feedback  
**Target Platform**: Mobile-first gameplay (Android/iOS) with desktop/editor parity  
**Project Type**: Single-player Godot game feature extension  
**Performance Goals**: Preserve current gameplay responsiveness and 60 fps while evaluating at most 9 input slots per craft attempt  
**Constraints**: Seed recipes only; deterministic outcomes; no legacy structure/build migration; touch targets must be >= 48x48 px; token consumption must happen only after successful inventory insert  
**Scale/Scope**: 5 single-token seed recipes + 10 dual-token seed recipes, one 3x3 grid interaction surface, targeted regression-safe updates to existing crafting flow

## Phase 2: Implementation Planning Outline

Ordered workstreams for `/speckit.tasks` generation:

1. **Grid Input Surface (3x3)**
   - Implement or adapt crafting UI grid to exactly 9 slots.
   - Enforce per-slot occupancy (zero or one token).
   - Ensure mobile slot touch target >= 48x48 px.
2. **Seed Recipe Resolution Core**
   - Build normalized input extraction from occupied slots.
   - Resolve only Phase 1 seed recipes (single/dual token mappings).
   - Treat all non-seed and legacy structure combinations as non-matching.
3. **Attempt Outcome and Consumption Ordering**
   - Evaluate preconditions (unlock gates, recipe validity, inventory capacity) before consumption.
   - On success: add exactly one seed item to plant inventory, consume only participating tokens, clear consumed slots.
   - On inventory full for valid recipe: block completion, keep tokens in grid, emit inventory-full feedback.
4. **Failure Feedback Wiring**
   - Provide clear user-facing messages for empty grid, invalid combination, locked Ku, and inventory full.
   - Keep failure states non-destructive (no unintended token removal).
5. **Validation and Regression Coverage**
   - Add/extend GUT tests for all Phase 1 mappings, position-insensitivity permutations, inventory-full blocking, and consumption semantics.
   - Add manual verification checklist for UI touch target and feedback readability.
   - Confirm no structure/build crafting or migration behavior changes in this phase.
