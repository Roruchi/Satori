# Implementation Plan: Craft Mode Building Placement and Build Mode Retirement

**Branch**: `020-craft-mode-buildings` | **Date**: 2026-04-03 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/020-craft-mode-buildings/spec.md`

## Summary

Retire the existing Build mode and move all building creation into craft-mode pattern resolution. Building crafts are discovered through successful pattern outcomes, produce inventory-stored building items in a unified 8-slot inventory, stack only by exact building type (cap 99, rollover to new stack when free slot exists), fail with no consumption when inventory insertion is impossible, and are placed through explicit inventory-driven confirm/cancel placement sessions supporting both one-tile and multi-tile footprints. Regular non-building tile placement remains immediate and does not add a confirm step.

## Phase 2: Implementation Strategy (for /speckit.tasks)

1. **Retire Build Mode UI and Input Branches**
   - Remove Build tab/state from HUD mode model and tab presentation.
   - Eliminate build-mode-only input pathways in placement/controller flow.
   - Preserve Plant/Craft/Interact/Codex behavior unaffected.

2. **Add Building Pattern Craft Resolution**
   - Extend craft resolver to evaluate building patterns (>=3 occupied slots).
   - Keep pattern normalization deterministic and mode-consistent.
   - Emit clear feedback for success/no-match/locked/failure states.

3. **Unify Inventory to 8 Slots with Building Stack Rules**
   - Update inventory capacity to 8 slots.
   - Support exact-type building stacking only.
   - Enforce stack cap 99 and rollover into free slot when available.
   - Fail craft non-destructively if insertion is impossible.

4. **Discovery and Failure Semantics Enforcement**
   - Record discovery only on successful craft output.
   - Ensure failed inventory-full craft does not consume ingredients or unlock discovery.
   - Keep codex/discovery notifications in sync with successful outcomes.

5. **Inventory-Driven Placement Session**
   - Add building selection-driven placement session state.
   - Validate one-tile and multi-tile footprints against world bounds/occupancy.
   - Keep explicit confirm/cancel contract with correct inventory mutation.
   - Preserve immediate non-building tile placement (no confirm requirement) to avoid regressions.

6. **Visual Feedback and Preview Integration**
   - Surface invalid-footprint and inventory/full-stack feedback clearly.
   - Update hover/preview rendering paths for building footprints.
   - Preserve interaction clarity on mobile and desktop inputs.

7. **Regression and Reference Validation**
   - Update/replace build-mode regressions with craft-mode building regressions.
   - Add tests for stacking cap, rollover behavior, and no-consumption failure.
   - Add tests for confirm/cancel placement invariants and footprint blocking.
   - Update `specs/master/recipes.md` if unlock/discovery catalog content changes; keep `tests/unit/test_recipes_catalog.gd` green.
